Створення і редагування задачника (меню Workbooks, посилання "Новий задачник")
    В полі Title назва задачника.
    В полі Problems ідентифікатори і назви задач.
    В полі Open Time дата/час, коли задачник відкриється і задачі можна буде вирішувати.
    В полі Open Minutes кількість хвилин, на протязі яких задачник буде відкритий. Якщо кількість дорівнює нулю або відсутня, задачник буде відкритий завжди.
    В полі Username Sample регулярний вираз, що обирає користувачів, яким буде видно відкритий задачник.
    Якщо вираз відсутній, задачник побачать усі.
    Докладніше про поле Problems:
    -- задачі додаються і видаляються з задачника за допомогою списку задач з чекбоксами внизу сторінки. наявний зміст задачника можна спостерігати в полі Problems.
    -- рядки в полі Problems можна міняти місцями і видаляти. Послідовність рядків є послідовністю задач в задачнику;
    -- щоб задачу можна було вирішувати лише в VS Code, треба поставити зірочку нарикінці рядку задачі;
    -- в базі даних зберігається саме змість поля Problems, а не стан чекбоксів в списку задач.
    Задачник для контрольних заходів
    -Tакий задачник містить лише одну задачу.
    -Cтудент повинен вперше відкрити задачу на протязі однієї хвилини з моменту відриття задачника.

Створення і редагування задач (меню Problems, посилання "Нова Задача")
    В полі title назва задачі.
    В полі attr атрибути задачі у формі aaa/bbb/ccc...
    В полі author им'я володара задачі. Лише володар має право змінювати або видаляти задачу. Користуватися задачею можуть усі.
    В полі cond умова задачі. 
    В полі view те, що буде знахоитись у вікни коду, коли користувач відкриє задачу для вирішення.
    В полі codе програма, яка тестує користувацькі вирішення. Необхідною частиною програми є авторське вирішення, яке оточується маркерними рядками: 
    - //BEGIN і //END для мов C# i Javascript
    - #BEGIN і #END для мови Python.
    Рекомендована схема перевірки:
    - if (хибна_умова_1) throw new Exception("Wrong. Необов'язкове пояснення_1");
    - ...
    - if (хибна_умова_N) throw new Exception("Wrong. Необов'язкове пояснення_N");
    - return 0;
    При збереженні задачі перевіряється, що аворське вирішення успішно проходить тестування. Якщо ні, оновлення не зберігаються.

Вирішення задач (меню Solving)
    Відкривається сторінка, на якій перелічені всі задачники, доступні користувачу в даний момент часу. Під назвою задачника назви задач, з яких він складається. Задачі, які вже вирішені, користувачу не показують.
    Назва задачі є посиланням, за яким відкривається вікно вирішення задачі. У вікні вирішення користувач пише код вирішення і натиском кнопки "Перевірити" відправляє код на сервер для первірки.


Перегляд рішень (меню Workbooks, клік по задачнику, клік по імені вірішувача)
    Відкривається сторінка, на якій видно трасу вирішення задачі.
